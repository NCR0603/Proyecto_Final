Arquitectura WEB Cliente Servidor

En esta imagen podemos observar la arquitectura WEB cliente servidor, la misma consta de los siguientes componentes:


Cliente: Es cualquier dispositivo con acceso a una red que realiza una solicitud mediante la red ya sea intranet o internet a un servidor.

Red: un conjunto de dispositivos interconectados entre sí a través de un medio, que intercambian información y comparten recursos.

Servidor: Un servidor basado en software es un programa que ofrece un servicio especial que otros programas denominados clientes (clientes) pueden usar a nivel local o a través de una red. El tipo de servicio depende del tipo de software del servido.


Diagrama MVC

En esta imagen podemos ver el modelo este es un patrón de arquitectura de software, que separa los datos y principalmente lo que es la lógica de negocio de una aplicación de su representación y el módulo encargado de gestionar los eventos y las comunicaciones


Arquitectura WEB Cliente Servidor Multicapa

En esta imagen podemos apreciar la arquitectura cliente servidor, pero en esta ocasión con la particularidad de que cada capa se subdivide en subcapas, estos subniveles se componen de la siguiente manera:


Vista: Nivel de cliente:componentes de interfaz del lado del cliente.

Nivel de lógica de presentación:programas del lado del servidor para generar páginas web.


Controlador: Nivel de lógica de negocio: modela los objetos de negocio. Nivel de web: el servidor web.
